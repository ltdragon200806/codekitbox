# Qiita / Zenn Article Draft

## Title

JWT の中身を安全に確認するためのブラウザツールを作った

## Body

API の開発やデバッグでは、JWT の `header` や `payload` をすぐ確認したい場面が
よくあります。しかし、本番環境のトークンを知らないオンラインサービスへ送信
するのは避けたいところです。

そこで、入力内容をブラウザ内で処理する小さなツール集 **CodeKitBox** を作り
ました。

https://codekitbox.com/ja/tools/jwt-decoder/

## JWT デコード時の注意点

JWT をデコードすると、`iss`、`aud`、`sub`、`exp` などのクレームを確認できます。
ただし、デコードできることと、トークンが正しいことは別です。

- デコードは署名検証ではない
- `exp` や `nbf` は通常 Unix 時刻の秒単位
- 本番トークンを共有しない
- アプリケーション側で署名とクレームを検証する

CodeKitBox の JWT デコーダーは、確認用としてヘッダーとペイロードをブラウザ内
で表示します。トークンの有効性を証明するものではありません。
