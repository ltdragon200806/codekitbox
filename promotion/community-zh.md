# 掘金文章发布稿

## 推荐标题

我做了一个尽量不上传数据的在线工具箱：CodeKitBox

## 正文

平时开发、学习或者处理资料时，我经常会遇到一些很小、但又必须马上解决的任务：
临时格式化一段 JSON、查看 JWT 里面的声明、转换 Unix 时间戳、生成二维码，或者
清理一份包含重复内容的文本列表。

这些任务单独看都不复杂，专门为它们安装软件又显得有些麻烦，于是很多时候我都会
打开搜索引擎，临时找一个在线工具。久而久之，我发现这个过程也有一些让我不太满意
的地方：部分网站广告很多，有些需要注册账号，还有一些工具并没有说明输入内容会被
如何处理。

因此，我做了一个免费在线工具箱 **CodeKitBox**：

https://codekitbox.com/

它目前包含 22 个常用工具，不需要注册账号，打开页面就可以使用。对于 JSON、JWT、
普通文字等常见输入，工具会尽可能直接在浏览器本地完成处理，不需要把内容发送到
应用服务器。

## 目前有哪些工具

第一版主要覆盖了开发过程中常见的转换、格式化和生成需求：

- Base64、URL 和 HTML 实体编码与解码
- JWT 解码、Unix 时间戳转换、JSON 格式化与压缩
- CSV 与 JSON 转换、URL 结构解析
- 图片压缩、图片尺寸调整、图片格式转换
- UUID、SHA 哈希、二维码和强密码生成
- 正则表达式测试、文本对比、大小写转换和文本清理
- 百分比计算、随机数生成、字数与字符统计

其中一些工具偏向开发者，例如 JWT 解码和 JSON 格式化；另外一些工具则更适合日常
使用，例如二维码、密码、百分比和字数统计。我希望它最终不只是“程序员工具箱”，
也能解决普通用户经常遇到的小问题。

## 为什么强调在浏览器本地处理

在线工具非常方便，但我们有时会不经意地把真实数据粘贴进去，例如接口返回内容、
配置片段、日志、访问令牌或者还没有公开的文字。

CodeKitBox 是一个静态网站，大部分功能直接使用浏览器提供的 JavaScript 和 Web API
完成。比如格式化 JSON、转换 Base64、生成密码和统计文字时，输入内容不需要上传到
应用服务器。

当然，“本地处理”并不代表可以忽略安全常识。即使使用本地运行的工具，我仍然建议
不要随意粘贴生产环境中的敏感令牌、密码和个人信息。以 JWT 为例，能够解码 Header
和 Payload 并不代表令牌有效，更不代表已经完成签名验证。

## 不只是放一个输入框

制作网站的过程中，我发现仅仅提供一个可以点击的按钮还不够。很多用户真正需要的，
是知道这个工具适合做什么、不能做什么，以及遇到错误时应该从哪里检查。

因此，我为 Base64、JWT、JSON、二维码和强密码等主题制作了独立使用指南，并在工具
页面中加入常见用途、容易踩的坑、隐私说明和常见问题。

例如：

- Base64 是编码方式，不是加密方式；
- JWT 解码不等于签名验证；
- 标准 JSON 不支持单引号和尾随逗号；
- 二维码生成后，应当使用真实手机测试最终尺寸；
- 密码的长度、随机性和唯一性通常比复杂的字符替换更重要。

这些内容可能不算高深，但能够帮助第一次接触相关格式的人少走一些弯路。

## 技术实现

CodeKitBox 没有使用大型前端框架，主要由 HTML、CSS 和原生 JavaScript 构成，并部署
在 Cloudflare 上。工具页面和使用指南通过本地脚本生成，以便统一维护英文、简体中文
和日文页面。

为了让搜索引擎更容易理解不同语言版本，我还添加了独立页面地址、Canonical、
hreflang、Sitemap 和结构化数据。项目源代码已经公开在 GitHub：

https://github.com/ltdragon200806/codekitbox

目前项目仍然很小，代码和设计都有继续改进的空间。公开源码也是希望能够获得更具体
的建议，而不是只根据自己的使用习惯做决定。

## 接下来准备做什么

网站刚刚上线，现在最重要的事情不是无限增加工具数量，而是观察哪些工具真的有人
使用，并把它们做得更顺手。

目前计划优先考虑：

1. 为二维码工具增加下载和复制功能；
2. 为百分比计算器增加折扣、涨跌幅和小费模式；
3. 改进 JSON 错误定位和格式化体验；
4. 根据反馈补充更多实用指南；
5. 继续检查移动端体验和无障碍使用。
